# C++基础相关

## const/constexpr/mutable关键字

**const常见用法**
* **修饰变量**：变量不可变
* **修饰指针**：
  * `const int* p` 或 `int const* p`：指向的对象是 const（不能通过 p 改 *p），p 本身可变
  * `int* const p`：指针本身是 const（不能改 p），指向对象可变
  * `const int* const p`：指针和指向对象都 const
* **修饰引用**：`const T&`：可绑定到左值/右值/临时对象；常用来避免拷贝并保持只读
* **修饰成员函数**：承诺不修改成员。此时this指针的类型是`const T*`，因此不能改非`mutable`成员。
  * 函数重载：编译器会把有const和无const修饰的函数当成两个不同的函数看待。const对象默认调用const函数，非const对象默认调用非const函数
* **修饰函数参数/返回值**：如果按值传参或按值返回，加const没有意义。如果按引用传参或按引用传递返回值，加const表示不可变。

**修改const变量**
可以通过某种手段绕过变量的const限制，比如`const_cast`，或者用指针指向这块内存地址，然后修改指针指向的值。但是所有对const对象的修改都是未定义行为，因为编译器在编译的时候可能会对const做编译优化。此外，const变量有可能会被放在.rodata分区，也就是说操作系统可能会把const变量放在内存的只读区域，对这一块内存做修改会导致`runtime memory violation`问题

**mutable关键字**
* 修饰变量：mutable关键字修饰的对象在const函数中也可以被修改。成员函数有const修饰时无法改变成员变量。但是我如果想用一个变量统计某个const的函数的调用次数，这时候就需要用到mutable对象了。
```cpp
struct Cache {
	mutable int hits{0};
	int get() const { ++hits; return 42; }
};
```
* 修饰lambda：lambda表达式内部不能修改通过值捕获的变量，但是用mutable修饰的lambda表达式，就可以绕过该限制。
```c++
{
    int a = 0;
    auto lambda = [=]()mutable{a=2;};
    // auto lambda = [=](){a=2;};  // 编译失败
}
```

**constexpr / const 的区别**

* `const`：只读语义，不一定是编译期常量。
* `constexpr`：要求可在编译期求值（在满足条件时）。


**Constexpr关键字**
* **修饰变量**：编译期常量，可在编译期求值
* **修饰函数**：`constexpr` 函数在“参数是常量表达式”时可在编译期求值，否则就当普通函数运行期求值。
```cpp
constexpr int add(int a, int b) { return a + b; }

constexpr int x = add(1, 2); // 编译期
int y = add(3, 4);           // 运行期
```

## 指针与引用

**指针（Pointer）的用法**
* **表达“地址/可选对象”**：指针可以为空（`nullptr`），因此很适合表达“可能没有对象”。
* **可变指向**：同一个指针变量在生命周期内可以指向不同对象。
* **间接访问**：通过 `*p` 访问对象、`p->m` 访问成员。
* **与数组/内存块配合**：指针可以参与指针算术，但语义只在“同一数组/同一连续内存块范围内”才有定义；跨对象做指针运算是未定义行为。
* **与资源管理分离**：裸指针只表达“位置”，不表达所有权；需要所有权语义应使用 `unique_ptr/shared_ptr` 或其他 RAII 封装。

**引用（Reference）的用法**
* **表达“别名/一定存在”**：引用必须绑定到某个对象上，语义上不允许为空。
* **参数传递**：
	* `T&` 表达“我要修改传入对象”。
	* `const T&` 表达“只读地使用传入对象”，并避免拷贝（也可绑定临时对象/右值）。
* **返回值**：可用于返回对象别名（例如容器的 `operator[]`），但必须确保被引用对象在调用者使用期间仍然存活。

**指针 vs 引用：区别与选择**
* **是否可空**：指针可空，引用语义上不可空。
* **是否可重新绑定**：指针可改指向；引用一旦绑定就不再“改绑”（语言层面不可 reseat）。
* **接口设计常用习惯**：
	* “可选参数/可能缺失”常用 `T*` 或 `std::optional<std::reference_wrapper<T>>`。
	* “必传且非空”常用 `T&` / `const T&`。

**原理**
* **引用的实现细节**：很多 ABI/编译器会把引用实现为“隐藏指针”，但标准并不强制；你只能依赖其语义（别名、必须初始化、不可改绑）。
* **生命周期与悬空**：
	* 指针和引用都会悬空：指向/绑定的对象一旦销毁，后续解引用/访问就是未定义行为。
	* `const T&` 可以绑定临时对象，并触发“临时对象生命周期延长”规则；但并不是所有场景都会延长（例如把引用存到成员里/容器里就不成立）。
* **类型系统约束**：
	* `const T*`/`T* const`/`const T&` 等组合用于表达“能不能改对象/能不能改指向”。
	* 引用折叠、右值引用（`T&&`）更多用于移动语义与完美转发（与“指针/引用基础”是同一套引用规则的延伸）。
* **别名与优化（aliasing）**：引用/指针都会形成别名关系，编译器优化会受“可能别名”的约束；同时要注意 `const_cast` 等绕过 const 语义的操作可能引入未定义行为（尤其是修改本来就是 const 的对象）。

## static关键字

`static` 在 C++ 里有三类常见含义：**存储期（生命周期）**、**类成员归属**、（历史上）**链接属性**。

**1）函数内的static变量**
* **生命周期**：从第一次初始化后开始一直存活到程序结束（静态存储期）。
* **初始化时机**：第一次执行到声明处才初始化（懒初始化），之后复用同一份对象。
* **线程安全**：C++11 起，局部静态变量的初始化具有线程安全保证（只初始化一次）。
* **常见用法**：常用于实现“按需构造”的单例/缓存，也常被用来缓解静态初始化顺序问题（SIOF），因为它把初始化延后到首次调用。

**2）类的static成员变量和函数**
* **static 成员变量**：属于“类本身”，所有对象共享同一份；不计入对象大小（对象里不存它）。
* **定义/ODR**：
  * C++17 之前：通常需要在类外提供一次定义（否则链接期可能报重复/缺失）。
  * C++17 起：可用 `inline static` 在类内直接定义并初始化，避免单独 `.cpp` 定义。
* **static 成员函数**：没有 `this` 指针，因此不能直接访问非 static 成员（除非通过传入对象/指针/引用）。

**4）对象模型与内存布局（理解层面）**
* static 成员变量独立于对象存在：对象内存布局不包含它。
* 访问 static 成员变量本质是访问一个全局/静态存储期的符号地址；编译器会生成相应取址/访问代码。

## C++新特性

下面按版本把“最常被问、最常用”的新特性列一遍（面试复习向）。

**C++11**
* 右值引用、移动语义、完美转发：`T&&`、`std::move`、`std::forward`
* `auto`、`decltype`
* 范围 for：`for (auto& x : vec)`
* lambda：`[capture](args){...}`
* 智能指针：`unique_ptr`、`shared_ptr`、`weak_ptr`
* 并发库：`std::thread`、`mutex`、`condition_variable`、`atomic`
* `constexpr`（初版能力较弱）
* 初始化列表、统一初始化 `{}`

**C++14**
* 泛型 lambda：`[](auto x){...}`
* `constexpr` 放宽（能写更复杂的逻辑）
* `std::make_unique`

**C++17**
* 结构化绑定：`auto [a,b] = pair;`
* `if constexpr`
* 折叠表达式（variadic）：`(args + ...)`
* `std::optional`、`std::variant`、`std::any`
* `std::string_view`
* inline variables：`inline static`（解决静态成员定义问题）
* 并行算法（部分实现支持）：`<execution>`

**C++20**
* Concepts：`template <std::integral T>`（约束模板参数）
* Ranges：管道式视图（需要编译器/库支持）
* 协程 coroutine（语言级）：`co_await/co_yield/co_return`
* `std::jthread`、`stop_token`（更现代的线程管理）
* `consteval`、`constinit`
* `std::span`
* 三路比较 `<=>`
* 模块（Modules，生态推进中，工程落地看编译器/构建系统）

**C++23**
* `std::expected`（更结构化的错误返回）
* `std::print` / `std::println`（实现支持度依赖标准库版本）
* Ranges 进一步补强
